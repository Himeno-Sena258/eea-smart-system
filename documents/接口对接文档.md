# 工程教育专业认证智能服务系统 - 接口对接文档

## 1. 文档说明

本文档用于前端与 Spring Boot 后端进行接口联调，接口设计依据项目功能需求、功能点截图以及 `documents/数据库.md` 中的数据库设计整理。

当前数据库文档中的核心表以 `sys_organization`、`sys_role`、`sys_user`、`class_info`、`student_info`、`program_scheme`、`grad_requirement`、`grad_indicator_point`、`course`、`course_objective`、`course_obj_indicator_map`、`teaching_class`、`teaching_class_student`、`assessment_method`、`assessment_item`、`student_score`、`evidence_material`、`course_attainment`、`continuous_improvement`、`survey_questionnaire`、`survey_answer`、`report`、`report_section`、`report_data_source`、`sys_audit_log` 等为准。若需求截图中的表名与数据库文档不完全一致，接口命名保持业务语义稳定，后端实现时按数据库文档映射。

## 2. 基础约定

### 2.1 接口前缀

```text
/api/v1
```

### 2.2 请求格式

- 普通请求：`Content-Type: application/json`
- 文件上传：`Content-Type: multipart/form-data`
- 批量导入：优先使用 `.xlsx` 文件，通过 `multipart/form-data` 上传
- 日期时间格式：`yyyy-MM-dd HH:mm:ss`
- 金额、分数、权重等小数：使用数字类型，权重范围通常为 `0.00 ~ 1.00`

### 2.3 登录校验约定

本项目后端采用简单登录校验方式，不使用 JWT。登录接口根据账号、密码查询数据库，校验通过后返回当前用户信息和角色信息。

前端可将当前登录用户信息保存到本地状态或浏览器存储中，用于页面路由守卫和按钮权限判断。退出登录时清除前端保存的用户信息即可。

若后端需要判断当前操作人，可在请求参数中显式传递 `operatorId`、`createdBy` 或 `userId`，具体字段以对应接口说明为准。

### 2.4 统一响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "traceId": "20260716190000123",
  "timestamp": "2026-07-16 19:00:00"
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | number | 业务状态码，`0` 表示成功 |
| `message` | string | 响应提示信息 |
| `data` | any | 响应数据 |
| `traceId` | string | 请求追踪 ID，用于排查问题 |
| `timestamp` | string | 服务端响应时间 |

### 2.5 分页响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "records": [],
    "pageNum": 1,
    "pageSize": 10,
    "total": 0,
    "pages": 0
  },
  "traceId": "20260716190000123",
  "timestamp": "2026-07-16 19:00:00"
}
```

分页查询通用参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `pageNum` | number | 否 | `1` | 当前页 |
| `pageSize` | number | 否 | `10` | 每页数量 |
| `keyword` | string | 否 | - | 搜索关键词 |

### 4.2 用户注册

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/api/v1/auth/register` | 用户注册 |

请求示例：

```json
{
  "username": "2024001001",
  "password": "123456",
  "confirmPassword": "123456",
  "realName": "张三",
  "email": "student@example.com",
  "phone": "13800000000",
  "roleCode": "STUDENT",
  "orgId": 10,
  "studentNo": "2024001001",
  "classId": 1
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `username` | string | 是 | 登录账号，教师可用工号，学生可用学号 |
| `password` | string | 是 | 登录密码 |
| `confirmPassword` | string | 是 | 确认密码 |
| `realName` | string | 是 | 真实姓名 |
| `email` | string | 否 | 邮箱 |
| `phone` | string | 否 | 手机号 |
| `roleCode` | string | 是 | 角色编码：`ADMIN`、`DIRECTOR`、`COORDINATOR`、`INSTRUCTOR`、`STUDENT` |
| `orgId` | number | 否 | 所属组织 ID |
| `studentNo` | string | 学生必填 | 学号 |
| `classId` | number | 学生必填 | 行政班级 ID |

响应数据：

```json
{
  "id": 1001,
  "username": "2024001001",
  "realName": "张三",
  "roleCode": "STUDENT",
  "status": 1,
  "createdAt": "2026-07-16 19:00:00"
}
```

实现说明：

- 注册时后端需对密码进行 BCrypt 加密后写入 `sys_user.password`。
- 学生注册时同时写入 `sys_user`、`sys_user_role`、`student_info`。
- 教师或管理员账号也可由管理员在用户管理模块创建，是否开放自主注册由系统配置决定。

## 6. 课程与支撑关系模块

### 6.1 课程-指标点支撑矩阵

对应控制器：`MatrixController`  
对应数据表：`course_obj_indicator_map`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/v1/courses/{courseId}/indicator-matrix` | 查询课程支撑指标点矩阵 |
| `PUT` | `/api/v1/courses/{courseId}/indicator-matrix` | 保存课程支撑指标点矩阵 |
| `DELETE` | `/api/v1/courses/{courseId}/indicator-matrix` | 清空课程支撑关系 |

保存请求：

```json
{
  "items": [
    {
      "courseObjectiveId": 1,
      "indicatorPointId": 10,
      "weight": 0.4
    },
    {
      "courseObjectiveId": 2,
      "indicatorPointId": 11,
      "weight": 0.6
    }
  ]
}
```

校验规则：

- `courseObjectiveId` 必须属于当前课程。
- `indicatorPointId` 必须属于当前课程所在培养方案。
- 同一 `courseObjectiveId + indicatorPointId` 不允许重复。
- `weight` 取值范围为 `0.01 ~ 1.00`。

| `GET` | `/api/v1/self-evaluation/sections/{sectionId}/data-sources` | 查询章节自动填充数据源 |
| `PUT` | `/api/v1/self-evaluation/sections/{sectionId}/data-sources` | 保存章节自动填充数据源 |
| `POST` | `/api/v1/self-evaluation/reports/{id}/auto-fill` | 根据系统数据自动填充 |
| `GET` | `/api/v1/self-evaluation/reports/{id}/export` | 导出报告 |

创建报告请求：

```json
{
  "schemeId": 1,
  "title": "软件工程专业工程教育认证自评报告",
  "version": "V1.0",
  "createdBy": 20
}
```

报告基础信息响应：

```json
{
  "id": 1,
  "schemeId": 1,
  "title": "软件工程专业工程教育认证自评报告",
  "version": "V1.0",
  "status": 0,
  "createdBy": 20,
  "createdAt": "2026-07-16 19:00:00",
  "updatedAt": "2026-07-16 19:00:00"
}
```

新增章节请求：

```json
{
  "sectionCode": "1.1",
  "title": "学生",
  "content": "",
  "assignedTo": 21
}
```

修改章节内容请求：

```json
{
  "sectionCode": "1.1",
  "title": "学生",
  "content": "<p>本章节说明学生来源、培养过程与支持条件。</p>",
  "assignedTo": 21,
  "status": 1
}
```

保存章节数据源请求：

```json
{
  "dataSources": [
    {
      "sourceType": "ATTAINMENT",
      "sourceKey": "scheme:1",
      "autoFillConfig": {
        "chartType": "radar",
        "title": "毕业要求达成度雷达图"
      }
    },
    {
      "sourceType": "SURVEY",
      "sourceKey": "survey:10",
      "autoFillConfig": {
        "summaryType": "table"
      }
    }
  ]
}
```

字段说明：

| 字段 | 数据库字段 | 说明 |
| --- | --- | --- |
| `schemeId` | `report.scheme_id` | 关联培养方案 ID |
| `version` | `report.version` | 报告版本 |
| `status` | `report.status` / `report_section.status` | 报告或章节状态 |
| `sectionCode` | `report_section.section_code` | 章节编号 |
| `assignedTo` | `report_section.assigned_to` | 章节负责人 |
| `sourceType` | `report_data_source.source_type` | 自动填充来源类型 |
| `sourceKey` | `report_data_source.source_key` | 数据来源标识 |
| `autoFillConfig` | `report_data_source.auto_fill_config` | 自动填充 JSON 配置 |

状态约定：

- 报告状态：`0`-编写中，`1`-审核中，`2`-已完成，`3`-已归档。
- 章节状态：`0`-未开始，`1`-编写中，`2`-已完成。
- 数据源类型：`ATTAINMENT`-达成度，`SURVEY`-问卷，`TABLE`-数据表格，`CHART`-图表。

## 15. 前后端联调顺序建议

1. 基础数据：专业、培养方案、毕业要求、指标点、课程。
2. 课程大纲：课程目标、指标点支撑矩阵、考核方式、考核细项。
3. 教学运行：教师、学生、教学班、学生名单。
4. 成绩过程：成绩录入、过程记录、佐证材料上传。
5. 计算分析：课程目标达成度、毕业要求达成度、不达标学生筛查。
6. 评价闭环：问卷调查、持续改进、自评报告。
