# 工程教育专业认证智能服务系统 - 角色功能与数据表规约说明书

---

## 一、 系统设计核心规约

1. **取消无门槛自主注册**：
   - 为保障认证数据的权威性与严谨性，系统完全关闭普通用户自主注册功能。
   - 所有学生学号、教师工号、行政班级归属及权限角色，均由 **系统管理员（`ADMIN`）** 通过后台统一开户或导入教务处官方 Excel 花名册生成。


---

## 二、 5 大角色功能菜单与数据表明细

### 1. 🛠️ 系统管理员 (`ADMIN`) — 全局运维与开户中心

> **职责**：负责学校组织架构建立、全校师生账号导入/开户、排课数据同步及系统安全审计，不参与具体教学业务成绩看板。

| 一级模块 | 二级功能菜单 | 表格展示核心字段 / 数据列明细 | 后端 API / 核心业务逻辑 |
| :--- | :--- | :--- | :--- |
| **开户与账号管理** | 1.1 师生账号批量导入 | Excel 预览表：`工号/学号`, `姓名`, `角色`, `所属专业/班级`, `手机号`, `校验结果` | 解析 Excel 批量写入 `sys_user`，自动生成默认密码并加密 |
| | 1.2 单个账号新增与开户 | 用户表：`用户ID`, `登录账号`, `真实姓名`, `分配角色`, `所属组织`, `账号状态` | 管理员手动创建教师/学生账号，校验账号唯一性 |
| | 1.3 密码重置与状态管控 | 账号列表：`账号`, `姓名`, `状态(启用/禁用)`, `最近登录时间`, `操作(重置密码/禁用)` | 重置初始密码、封禁违规账号 |
| **基础数据维护** | 1.4 组织机构架构管理 | 架构树/表：`机构ID`, `机构名称`, `父级ID`, `类型(学院/专业/行政班)`, `建立时间` | 建立学院 $\rightarrow$ 专业 $\rightarrow$ 行政班级三级组织树 |
| | 1.5 排课与选课关系导入 | 排课表：`学期`, `课程代码`, `课程名称`, `主讲教师工号`, `教学班名称`, `选课学生数` | 导入/同步教务排课数据，建立 `teaching_class_student` |
| **安全审计** | 1.6 系统操作审计日志 | 日志表：`日志ID`, `操作时间`, `操作账号`, `操作类型(改分/删方案)`, `模块`, `IP地址` | 分页查询 `sys_audit_log`，只读不可篡改 |

### 2. 🎓 专业负责人 (`DIRECTOR`) — 专业建设与认证总指挥

> **职责**：负责培养方案修订、毕业要求 12 大项及 30+ 指标点拆解、课程支撑矩阵设置（强校验和=1.000）、全专业达成度驾驶舱表及自评报告编写派发。

| 一级模块 | 二级功能菜单 | 表格展示核心字段 / 数据列明细 | 后端 API / 核心业务逻辑 |
| :--- | :--- | :--- | :--- |
| **培养方案管理** | 2.1 培养方案版本管理 | 方案列表：`方案ID`, `版本名称(如2024版)`, `专业`, `状态(草稿/发布/归档)`, `建立时间` | 版本 CRUD，发布后禁止修改指标点 |
| | 2.2 毕业要求与指标点拆解| 明细表：`大项编码(GR1~GR12)`, `指标点编码(1.1, 1.2)`, `指标点内容`, `目标合格线(0.680)` | 维护 12 大项及二级指标点文本与合格阈值 |
| | 2.3 课程支撑矩阵配置 | **二维矩阵大表**：`横轴: 毕业要求指标点`, `纵轴: 专业课程`, `单元格: 支撑权重(0.01~1.00)`, `列底: 权重和(1.000)` | 矩阵配置，**后端强校验 $\sum W_{support} = 1.000$**，不等于 1.0 拦截抛错 |
| **专业达成度** | 2.4 指标点达成度汇总表 | 达成度表：`指标点编码`, `指标点内容`, `标准目标值(0.680)`, `全专业实测达成值`, `差值`, `状态(合格/预警)` | 触发达成度引擎计算，查询 `grad_indicator_attainment` 固化数据 |
| | 2.5 薄弱指标点预警表 | 预警表：`未达标指标点`, `实测达成值(<0.680)`, `主要支撑课程`, `责任教师`, `差值幅度` | 自动筛选不达标指标点，反查并定位责任课程与教师 |
| **间接评价与改进**| 2.6 问卷发布与满意度表 | 统计表：`问卷类型(毕业生/用人单位)`, `指标点编码`, `回收问卷数`, `满意度均分`, `满意度占比(%)` | 设计并发布间接评价问卷，汇总统计得分 |
| | 2.7 专业持续改进记录 | 决策表：`改进年份`, `触发指标点`, `问题分析`, `培养方案/大纲调整决策`, `责任人` | 记录专业级持续改进历史闭环 |
| **自评报告协同** | 2.8 报告章节分派与套编 | 任务表：`章节编号(1.1, 1.2)`, `章节标题`, `指派负责教师`, `编写状态`, `绑定自动填充数据表` | 派发 7 大章节任务，自动插入达成度数值表格并一键导出 Word |

### 3. 👨‍💼 课程负责人 (`COORDINATOR`) — 课程大纲总设计师

> **职责**：负责课程大纲编制、课程目标（CO1~CO5）制定、考核占比配置、试题与 CO 强绑定映射、跨班级达成度对比分析。

| 一级模块 | 二级功能菜单 | 表格展示核心字段 / 数据列明细 | 后端 API / 核心业务逻辑 |
| :--- | :--- | :--- | :--- |
| **课程大纲设计** | 3.1 课程基本信息维护 | 课程表：`课程代码`, `课程名称`, `课程类别`, `学分`, `理论学时`, `实验学时`, `关联方案` | 维护课程元数据 |
| | 3.2 课程目标(CO)管理 | 目标表：`课程目标编码(CO1~CO5)`, `课程目标描述`, `对应支撑指标点`, `支撑权重` | 制定 3~5 个具体可衡量课程目标 |
| **考核方案设置** | 3.3 考核占比构成配置 | 考核构成表：`考核环节(期末/实验/平时)`, `在总分中占比权重(0.60/0.20/0.20)`, `权重和校验(1.00)` | 配置考核比例，校验总和等于 1.00 |
| | 3.4 试题/小项与CO映射表| 细项映射表：`考核环节`, `小项名称(如试卷第1题)`, `满分数值(如20.0)`, `强绑定的唯一CO` | 拆解考题/作业小项，强绑定唯一课程目标 $CO_k$ |
| **质量监控** | 3.5 跨教学班达成度对比表| 对比表：`教学班级`, `主讲教师`, `选课人数`, `CO1达成值`, `CO2达成值`, `CO3达成值`, `班级均分` | 横向对比不同老师、不同班级的达成度偏差 |
| | 3.6 课程年度质量报告 | 报告表：`学期`, `课程名称`, `总教学班数`, `总人数`, `整体达成度实测均值`, `报告状态` | 汇总课程数据生成年度课程质量报告 |

---

### 4. 👨‍🏫 授课教师 (`INSTRUCTOR`) — 教学一线执行者

> **职责**：负责教学班花名册管理、按题目/作业小项录入学生成绩（网络表格/Excel）、生成成绩单、班级 CO 达成度一键计算、上传高/中/低归档样品。

| 一级模块 | 二级功能菜单 | 表格展示核心字段 / 数据列明细 | 后端 API / 核心业务逻辑 |
| :--- | :--- | :--- | :--- |
| **班级教学** | 4.1 授课班级学生花名册 | 花名册表：`学期`, `教学班名称`, `课程名称`, `学号`, `学生姓名`, `行政班`, `选课状态` | 查询本班学生名单，导出空白录分 Excel 模板 |
| **成绩录入与计算**| 4.2 细项成绩网格录入表 | 网格表：`学号`, `姓名`, `试卷第1题(CO1)`, `试卷第2题(CO2)`, `实验1(CO2)`, `平时1(CO1)...` | 批量接收小项分数（支持 Excel 导入），校验分数不能超满分 |
| | 4.3 班级总评成绩单表 | 成绩单表：`学号`, `姓名`, `平时总分(20%)`, `实验总分(20%)`, `期末总分(60%)`, `综合总评分`, `是否及格` | 系统按权重公式自动计算总评并存入 `student_course_score` |
| **达成度与改进**| 4.4 班级CO达成度计算表 | 达成度表：`课程目标(CO1~CO5)`, `支撑指标点`, `目标满分`, `班级平均分`, `直接达成度实测值`, `警戒线(0.680)` | 执行计算公式 $A(CO_k) = \frac{\sum W_m \cdot S_{avg}}{\sum W_m \cdot S_{max}}$，写入 `course_attainment` |
| | 4.5 班级教学改进反思表 | 反思表：`教学班级`, `低达成度CO`, `原因分析`, `拟采取改进措施`, `提交时间` | 对低于 0.680 的目标提交改进分析 |
| **佐证材料归档** | 4.6 认证样品归档表 | 样品表：`教学班级`, `考核环节(期末/实验)`, `样品档次(高-优秀/中-中等/低-不及格)`, `文件名`, `上传时间` | 上传试卷、作业、实验报告扫描件 PDF 并归档 |

### 5. 👨‍🎓 学生 (`STUDENT`) — 学习主体与评价者

> **职责**：查看个人各科小项及总分明细、查看个人毕业要求指标点达成数据、在线填写间接评价问卷。

| 一级模块 | 二级功能菜单 | 表格展示核心字段 / 数据列明细 | 后端 API / 核心业务逻辑 |
| :--- | :--- | :--- | :--- |
| **成绩与达成** | 5.1 课程小项分文明细表 | 明细表：`课程名称`, `考核环节`, `考核细项(如试卷第1题)`, `目标满分`, `个人得分`, `得分率(%)` | 查阅自己每门课每道题的实际得分与扣分项 |
| | 5.2 综合成绩总表 | 成绩表：`学期`, `课程代码`, `课程名称`, `平时分`, `实验分`, `期末分`, `综合总评分`, `是否及格` | 查询各科总评成绩与及格状态 |
| | 5.3 个人毕业要求达成表| 达成表：`指标点编码`, `指标点内容`, `合格标准线(0.680)`, `个人达成度实测值`, `达标状态` | 查询个人在 30+ 能力指标点上的实测数值与短板 |
| **课程与问卷** | 5.4 课程大纲与标准表 | 大纲表：`课程名称`, `学分`, `课程目标(CO1~CO5)`, `考核占比列举`, `对应支撑指标点` | 查阅课程学习目标与考核重点 |
| | 5.5 间接评价问卷填写 | 问卷表：`问卷名称`, `问卷类型`, `截止时间`, `提交状态`, `操作(在线填写)` | 查询未填问卷并提交答案 JSON |
