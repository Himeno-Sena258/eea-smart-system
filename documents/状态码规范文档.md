# 工程教育专业认证智能服务系统 - 状态码规范文档

## 1. 文档说明

本文档定义工程教育专业认证智能服务系统前后端接口的 HTTP 状态码、业务状态码、错误响应格式和前端处理规则。所有后端接口应统一使用本规范返回数据，便于前端 Axios 拦截器统一处理。

## 2. 响应格式

### 2.1 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "traceId": "20260716190000123",
  "timestamp": "2026-07-16 19:00:00"
}
```

### 2.2 失败响应

```json
{
  "code": 30001,
  "message": "参数校验失败",
  "data": {
    "errors": [
      {
        "field": "courseName",
        "message": "课程名称不能为空"
      }
    ]
  },
  "traceId": "20260716190000123",
  "timestamp": "2026-07-16 19:00:00"
}
```

### 2.3 字段说明

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `code` | number | 是 | 业务状态码，`0` 表示成功 |
| `message` | string | 是 | 给前端展示或调试的提示信息 |
| `data` | any | 否 | 成功数据或错误详情 |
| `traceId` | string | 是 | 请求链路追踪 ID |
| `timestamp` | string | 是 | 服务端响应时间 |

## 3. HTTP 状态码规范

HTTP 状态码表达请求在协议层面的处理结果，业务错误仍通过响应体中的 `code` 表达。

| HTTP 状态码 | 使用场景 | 前端处理建议 |
| --- | --- | --- |
| `200 OK` | 请求成功，包含业务成功或可预期业务失败 | 根据响应体 `code` 判断 |
| `201 Created` | 新增资源成功 | 可提示创建成功 |
| `204 No Content` | 删除成功且无响应体 | 可提示删除成功 |
| `400 Bad Request` | 请求格式错误、参数类型错误 | 展示错误信息 |
| `401 Unauthorized` | 未登录或登录校验失败 | 清理前端登录态并跳转登录页 |
| `403 Forbidden` | 已登录但无权限访问 | 展示无权限提示 |
| `404 Not Found` | 接口或资源不存在 | 展示资源不存在 |
| `405 Method Not Allowed` | 请求方法不支持 | 提示接口方法错误 |
| `409 Conflict` | 数据冲突，如编码重复、状态不允许操作 | 展示业务冲突原因 |
| `413 Payload Too Large` | 上传文件过大 | 提示文件大小限制 |
| `415 Unsupported Media Type` | 文件类型或请求格式不支持 | 提示格式不支持 |
| `429 Too Many Requests` | 请求过于频繁 | 提示稍后重试 |
| `500 Internal Server Error` | 服务端未预期异常 | 展示系统异常并记录 `traceId` |
| `503 Service Unavailable` | 服务不可用或维护中 | 提示稍后重试 |

## 4. 业务状态码总览

业务状态码采用五位数字：

```text
0       成功
1xxxx   通用错误
2xxxx   认证与权限错误
3xxxx   参数与数据校验错误
4xxxx   业务规则错误
5xxxx   数据库与数据状态错误
6xxxx   文件上传下载与导入导出错误
7xxxx   达成度计算错误
8xxxx   调查评价与自评报告错误
9xxxx   外部文件服务错误
```

## 15. 前端处理建议

### 15.1 Axios 拦截器处理规则

| 场景 | 判断条件 | 处理方式 |
| --- | --- | --- |
| 成功 | `code === 0` | 返回 `data` |
| 未登录 | HTTP `401` 或 `code` 为 `20001`、`20002`、`20003` | 清理前端登录态，跳转登录页 |
| 无权限 | HTTP `403` 或 `code` 为 `20004`、`20006` | 展示无权限提示 |
| 参数错误 | `3xxxx` | 展示校验信息，表单字段可逐项标红 |
| 业务冲突 | `4xxxx` | 展示业务错误提示 |
| 文件错误 | `6xxxx` | 展示上传或导入失败原因 |
| 达成度错误 | `7xxxx` | 引导用户补充课程目标、考核项或成绩数据 |
| 系统异常 | HTTP `500` 或 `1xxxx`、`5xxxx` | 展示系统异常，并保留 `traceId` |

### 15.2 表单校验错误展示

当后端返回 `30001` 时，`data.errors` 建议使用以下格式：

```json
{
  "errors": [
    {
      "field": "weight",
      "message": "权重必须在 0.01 到 1.00 之间"
    }
  ]
}
```

前端可根据 `field` 定位表单项并显示错误信息。

### 15.3 Excel 导入错误展示

当后端返回 `60008` 或 `60009` 时，`data.errors` 建议使用以下格式：

```json
{
  "totalRows": 100,
  "successRows": 96,
  "failedRows": 4,
  "errors": [
    {
      "rowIndex": 8,
      "field": "studentNo",
      "message": "学号不能为空"
    }
  ]
}
```

前端应展示导入汇总，并提供失败明细下载或复制能力。

## 16. 后端实现建议

### 16.1 全局异常处理

Spring Boot 建议使用 `@RestControllerAdvice` 统一处理异常：

- `MethodArgumentNotValidException` -> `30001`
- `MissingServletRequestParameterException` -> `30002`
- `MethodArgumentTypeMismatchException` -> `30003`
- `AccessDeniedException` -> `20004`
- `MaxUploadSizeExceededException` -> `60002`
- 自定义 `BusinessException` -> 对应业务码
- 其他未知异常 -> `10000`

### 16.2 业务异常结构

```java
public class BusinessException extends RuntimeException {
    private final int code;

    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }
}
```

### 16.3 日志要求

- 每次请求生成 `traceId`。
- 业务错误记录 `warn` 日志。
- 系统异常记录 `error` 日志，并包含 `traceId`、请求路径、请求参数、异常堆栈。
- 前端反馈问题时，应提供 `traceId` 方便后端定位。
