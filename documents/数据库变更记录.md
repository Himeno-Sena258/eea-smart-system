# 工程教育专业认证智能服务系统 - 数据库变更记录

本变更记录根据小组讨论决议整理，旨在说明数据库表结构的变更内容、新增模块的设计方案以及变更后的完整表结构，供系统开发、数据库维护和系统集成时参考。

---

## 1. 变更概述

本次变更包含以下内容：

*   移除 AI 基础设施配置模块（`ai_prompt_template` 表），原因：Spring AI 集成方案暂不实施，后续按需恢复。
*   新增自评报告协同模块（`report`、`report_section`、`report_data_source` 三张表），支撑专业负责人的自评报告协同编写功能。
*   新增系统审计日志表（`sys_audit_log`），支撑系统管理员的安全审计功能。

变更后，数据库共包含 **26** 张数据表，划分为以下 **10** 个子模块。

---

## 2. 变更后模块划分

1.  **组织与权限模块**：包含组织架构表（`sys_organization`）、系统角色表（`sys_role`）、用户基础信息表（`sys_user`）和用户角色关联表（`sys_user_role`）。
2.  **班级与学生模块**：包含行政班级信息表（`class_info`）和学生拓展信息表（`student_info`）。
3.  **培养方案与指标点**：包含人才培养方案表（`program_scheme`）、毕业要求大项表（`grad_requirement`）和毕业要求二级指标点表（`grad_indicator_point`）。
4.  **课程大纲与OBE矩阵**：包含课程信息表（`course`）、课程目标表（`course_objective`）和课程目标与指标点支撑矩阵（`course_obj_indicator_map`）。
5.  **教学运行与选课**：包含教学班级运行信息表（`teaching_class`）和教学班选课学生关联表（`teaching_class_student`）。
6.  **考核标准与得分录入**：包含课程考核评价方式表（`assessment_method`）、课程考核细小项表（`assessment_item`） and 学生得分明细表（`student_score`）。
7.  **材料归档与直接评价**：包含佐证材料归档表（`evidence_material`）、课程目标直接达成度计算结果表（`course_attainment`）和持续改进分析报告表（`continuous_improvement`）。
8.  **问卷调查与间接评价**：包含调查问卷表（`survey_questionnaire`）和问卷答案表（`survey_answer`）。
9.  **自评报告协同**：包含自评报告主表（`report`）、报告章节表（`report_section`）和章节数据自动填充配置表（`report_data_source`）。
10. **系统审计日志**：包含操作审计日志表（`sys_audit_log`）。

---

## 3. 删除表说明

### 3.1 AI 提示词配置表 (`ai_prompt_template`) — 已移除

原用于存储 Spring AI 智能代理的 System Prompt 模板，供大纲生成、改进建议等 Agent 调用时读取。

*   **移除原因**：AI Agent 集成方案技术难度较高，经小组讨论决定版本一期暂不实现该功能。

---

## 4. 新增表物理结构详细设计

### 4.1 自评报告主表 (`report`)
记录专业负责人创建的自评报告主文档，关联培养方案，支持版本管理和状态流转。
*   **主键**：`id` (自增)
*   **外键**：
    *   `fk_report_scheme` 指向 `program_scheme(id)`，删除培养方案时禁止级联删除。
    *   `fk_report_creator` 指向 `sys_user(id)`，删除用户时禁止级联删除。

| 字段名 | 数据类型 | 约束 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `bigint` | PK, AUTO_INCREMENT | | 报告ID |
| `scheme_id` | `bigint` | NOT NULL, FK | | 关联培养方案ID |
| `title` | `varchar(200)` | NOT NULL | | 报告标题 |
| `version` | `varchar(30)` | NOT NULL | | 版本号（如 `V1.0`, `V2.0`） |
| `status` | `tinyint` | NOT NULL | `0` | 状态: `0`-编写中, `1`-审核中, `2`-已完成, `3`-已归档 |
| `created_by` | `bigint` | NOT NULL, FK | | 创建人ID（专业负责人） |
| `created_at` | `timestamp` | NOT NULL | `CURRENT_TIMESTAMP` | 创建时间 |
| `updated_at` | `timestamp` | NOT NULL | `CURRENT_TIMESTAMP ON UPDATE` | 最后更新时间 |

---

### 4.2 报告章节表 (`report_section`)
将自评报告按大纲拆分为多个章节，支持多人协同编写与进度跟踪。
*   **主键**：`id` (自增)
*   **外键**：
    *   `fk_section_report` 指向 `report(id)`，删除报告时级联删除章节。
    *   `fk_section_user` 指向 `sys_user(id)`，删除用户时负责人字段置为 NULL。

| 字段名 | 数据类型 | 约束 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `bigint` | PK, AUTO_INCREMENT | | 章节ID |
| `report_id` | `bigint` | NOT NULL, FK | | 所属报告ID |
| `section_code` | `varchar(20)` | NOT NULL | | 章节编号（如 `1.1`, `1.2`, `2.1`） |
| `title` | `varchar(200)` | NOT NULL | | 章节标题 |
| `content` | `longtext` | | NULL | 章节正文内容（支持富文本） |
| `status` | `tinyint` | NOT NULL | `0` | 完成状态: `0`-未开始, `1`-编写中, `2`-已完成 |
| `assigned_to` | `bigint` | FK | NULL | 负责人ID |
| `updated_at` | `timestamp` | NOT NULL | `CURRENT_TIMESTAMP ON UPDATE` | 最后修改时间 |

---

### 4.3 章节数据自动填充配置表 (`report_data_source`)
配置报告章节中可自动从系统数据源（达成度计算、问卷调查等）动态填充的表格或图表。
*   **主键**：`id` (自增)
*   **外键**：`fk_rds_section` 指向 `report_section(id)`，删除章节时级联删除配置。

| 字段名 | 数据类型 | 约束 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `bigint` | PK, AUTO_INCREMENT | | 数据源ID |
| `section_id` | `bigint` | NOT NULL, FK | | 所属章节ID |
| `source_type` | `varchar(30)` | NOT NULL | | 来源类型: `ATTAINMENT`-达成度, `SURVEY`-问卷, `TABLE`-数据表格, `CHART`-图表 |
| `source_key` | `varchar(100)` | NOT NULL | | 数据来源标识（如教学班ID、问卷ID） |
| `auto_fill_config` | `text` | | NULL | 自动填充JSON配置（含查询参数、展示模板） |

---

### 4.4 系统操作审计日志表 (`sys_audit_log`)
记录所有用户的关键操作行为，支持按用户、时间维度查询，满足安全审计与操作回溯需求。
*   **主键**：`id` (自增)
*   **索引**：`idx_audit_user` (`user_id`)、`idx_audit_time` (`created_at`)
*   **注意**：该表不设外键约束，用户被删除后日志仍保留（`username` 字段冗余存储账号名）。

| 字段名 | 数据类型 | 约束 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `bigint` | PK, AUTO_INCREMENT | | 日志ID |
| `user_id` | `bigint` | | NULL | 操作用户ID |
| `username` | `varchar(50)` | | NULL | 操作账号名（冗余字段，用户被删除后仍可追溯） |
| `action` | `varchar(50)` | NOT NULL | | 操作类型: `LOGIN`, `CREATE`, `UPDATE`, `DELETE`, `EXPORT` |
| `target` | `varchar(100)` | | NULL | 操作对象（如表名或功能模块名） |
| `target_id` | `bigint` | | NULL | 操作对象记录ID |
| `detail` | `varchar(500)` | | NULL | 操作详情描述 |
| `ip_address` | `varchar(45)` | | NULL | 客户端IP地址 |
| `created_at` | `timestamp` | NOT NULL | `CURRENT_TIMESTAMP` | 操作时间 |

## 5. V3 数据库性能与结构优化变更记录

### 5.1 毕业要求指标点达成度结果表 (`grad_indicator_attainment`)
固化全专业各年级基于所有课程加权计算得出的毕业要求二级指标点达成度，避免全专业看板高频实时联查计算。
*   **联合唯一索引**：`uk_scheme_grade_ip` (`scheme_id`, `grade`, `indicator_point_id`)

### 5.2 毕业要求一级大项达成度结果表 (`grad_requirement_attainment`)
固化 12 条毕业要求大项达成度综合结果。
*   **联合唯一索引**：`uk_scheme_grade_req` (`scheme_id`, `grade`, `req_id`)

### 5.3 学生得分明细表优化 (`student_score`)
*   增加 `teaching_class_id` 冗余列，补充针对物理教学班维度的覆盖索引 `idx_tc_item` (`teaching_class_id`, `assessment_item_id`)，大幅提升按教学班录入与提取分数的查询性能。

### 5.4 学生综合总评成绩单表优化 (`student_course_score`)
*   补充联合唯一索引 `uk_student_tc` (`student_id`, `teaching_class_id`) 及物理外键，防止重复计算导致的脏数据。

### 5.5 系统审计日志字段扩容 (`sys_audit_log`)
*   将 `detail` 字段从 `varchar(500)` 扩展至 `text`，防止长 JSON 或操作详情文本截断。
