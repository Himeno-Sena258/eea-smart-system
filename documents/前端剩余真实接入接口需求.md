# 前端剩余真实接入接口需求

更新时间：2026-07-21

本文记录前端当前仍无法真实接入的功能点。范围仅包含已经在前端页面或服务中存在展示/交互，但后端接口、返回字段或数据库映射仍不足以支撑真实数据的部分。

## 1. 课程质量管理

### 1.1 清空单门课程指标矩阵

当前前端位置：课程质量管理页，课程目标与指标点支撑关系维护。

当前状态：
- 前端已有 `clearCourseIndicatorMatrix(courseId)` 占位函数。
- 后端已有培养方案矩阵保存接口：`POST /director/schemes/{schemeId}/matrix/save`。
- 后端没有“按单门课程清空矩阵关系”的明确接口。

建议补充接口：

```http
DELETE /courses/{courseId}/indicator-matrix
```

建议权限：
- `DIRECTOR`
- `ADMIN`

建议行为：
- 删除指定课程下所有 `course_objective_indicator` 或等价矩阵关系。
- 返回清空后的矩阵列表，或返回操作结果字符串。

数据库支撑：
- 数据库已有课程目标与指标点支撑关系表，能支撑删除。
- 目前缺的是后端接口。

## 2. 课程资源、课程大纲正文与教学内容

### 2.1 课程资源列表、上传、更新、删除、下载

当前前端位置：课程质量管理页的课程资源区域。

当前状态：
- 前端已有以下 service 函数：
  - `getCourseResourceList(courseId)`
  - `uploadCourseResource(courseId, payload)`
  - `updateCourseResource(id, payload)`
  - `deleteCourseResource(id)`
  - `downloadCourseResource(id)`
- 后端 controller 中未发现对应接口。
- 数据库迁移记录中也标注过 `course_resource` 表尚未创建。

建议补充接口：

```http
GET /courses/{courseId}/resources
POST /courses/{courseId}/resources
PUT /course-resources/{id}
DELETE /course-resources/{id}
GET /course-resources/{id}/download
```

建议权限：
- 查询：`DIRECTOR`、`COORDINATOR`、`INSTRUCTOR`、`STUDENT`
- 上传/更新/删除：`COORDINATOR`、`ADMIN`

建议数据库新增表：
- `course_resource`

建议字段：
- `id`
- `course_id`
- `resource_type`
- `description`
- `file_name`
- `file_path`
- `mime_type`
- `file_size`
- `preview_url`
- `uploaded_by`
- `uploaded_at`

数据库支撑：
- 当前数据库不完整，缺少 `course_resource` 表。
- 需要先补表，再补接口。

### 2.2 教学内容列表与保存

当前前端位置：课程质量管理页，教学内容/章节内容区域。

当前状态：
- 前端已有以下 service 函数：
  - `getTeachingContentList(courseId)`
  - `saveTeachingContents(courseId, payload)`
- 后端 controller 中未发现对应接口。
- 现有数据库是否有独立教学内容表需要后端确认；前端现有模型需要支持章节标题、学时、关联课程目标和排序。

建议补充接口：

```http
GET /courses/{courseId}/teaching-contents
PUT /courses/{courseId}/teaching-contents
```

建议权限：
- 查询：`DIRECTOR`、`COORDINATOR`、`INSTRUCTOR`、`STUDENT`
- 保存：`COORDINATOR`、`ADMIN`

建议请求结构：

```json
{
  "items": [
    {
      "id": 1,
      "title": "需求分析与建模",
      "hours": 8,
      "objectiveIds": [1, 2],
      "sortOrder": 1
    }
  ]
}
```

数据库支撑：
- 若当前没有教学内容表，需要新增。
- 若已有课程大纲正文 `content` 字段，只能支撑纯文本正文，不能支撑前端需要的结构化章节、学时和课程目标关联。

## 5. 课程负责人达成度分析

### 5.1 课程负责人课程达成度结果

当前前端位置：达成度分析页，课程负责人视图。

当前状态：
- 后端存在 `GET /coordinator/courses/{courseId}/attainment`。
- 但当前实现返回内容更接近课程目标列表，不是课程目标达成度或指标点达成度结果。
- 前端不能把该接口当作真实达成度分析数据接入。

建议调整接口：

```http
GET /coordinator/courses/{courseId}/attainment
POST /coordinator/courses/{courseId}/attainment/calculate
```

建议返回结构：

```json
[
  {
    "courseId": 1,
    "courseObjectiveId": 1,
    "objectiveCode": "CO1",
    "attainmentVal": 0.76,
    "warningThreshold": 0.68,
    "sampleCount": 42,
    "calculatedAt": "2026-07-21T10:30:00"
  }
]
```

数据库支撑：
- 如果已有 `course_attainment` 或等价表，可以支撑。
- 当前问题主要是接口实现语义不完整，返回值未对齐达成度结果模型。

## 6. 问卷题目明细与题目级统计

### 6.1 问卷题目结构

当前前端位置：问卷与评价页，问卷填写弹窗和统计弹窗。

当前状态：
- 后端已有：
  - `GET /surveys/{id}`
  - `GET /surveys/{id}/statistics`
  - `POST /surveys/{id}/answers`
- 统计接口当前只返回 `totalAnswers` 和原始答卷 `answers`。
- 后端以 `raw_answers_json` 保存答案，没有独立题目明细表或题目统计结构。
- 前端无法真实渲染题目列表、题目均分、选项分布、文本聚合。

建议补充接口：

```http
GET /surveys/{id}/questions
PUT /surveys/{id}/questions
GET /surveys/{id}/statistics
```

建议题目结构：

```json
[
  {
    "id": 1,
    "questionCode": "Q1",
    "title": "工程知识达成自评",
    "questionType": "SCORE",
    "options": [1, 2, 3, 4, 5],
    "sortOrder": 1
  }
]
```

建议统计结构：

```json
{
  "questionnaireId": 1,
  "title": "2024届毕业生达成度自评问卷",
  "totalAnswers": 170,
  "questions": [
    {
      "questionCode": "Q1",
      "title": "工程知识达成自评",
      "averageScore": 4.42,
      "distribution": [
        { "score": 5, "count": 96 },
        { "score": 4, "count": 54 },
        { "score": 3, "count": 17 },
        { "score": 2, "count": 3 }
      ]
    }
  ]
}
```

数据库支撑：
- 当前 `raw_answers_json` 能保存原始答案，但不适合可靠统计和题目维护。
- 建议新增问卷题目表和题目答案明细表，或至少在统计接口中解析并返回题目级聚合结果。

## 8. 持续改进聚合过滤与审核字段

### 8.1 专业/课程级持续改进聚合过滤

当前前端位置：持续改进页，专业负责人和课程负责人视图。

当前状态：
- 后端已有：
  - `GET /director/improvements?schemeId=`
  - `GET /coordinator/improvements?courseId=`
- 当前实现接收 `schemeId`、`courseId`，但没有实际过滤。
- 返回实体也不包含课程名、教学班名、教师名等展示字段。

建议调整接口：

```http
GET /director/improvements?schemeId={schemeId}
GET /coordinator/improvements?courseId={courseId}
```

建议返回字段：
- `id`
- `teachingClassId`
- `teachingClassName`
- `courseId`
- `courseName`
- `teacherId`
- `teacherName`
- `problemAnalysis`
- `improvementMeasures`
- `createdBy`
- `createdAt`

数据库支撑：
- 可通过 `continuous_improvement.teaching_class_id` 关联教学班、课程和教师。
- 当前主要缺接口过滤逻辑和返回 VO。

### 8.2 持续改进审核字段

当前状态：
- `V8__p0_gap_fixes.sql` 已为 `continuous_improvement` 增加：
  - `status`
  - `reviewed_by`
  - `reviewed_at`
  - `reviewer_comment`
  - `low_attainment_cos`
  - `cycle_label`
  - `follow_up_at`
  - `follow_up_result`
- 但当前 `ContinuousImprovement` 实体未暴露这些字段。
- 前端不能做审核状态、整改周期、跟踪结果展示。

建议补充接口：

```http
PUT /improvements/{id}/review
```

建议请求结构：

```json
{
  "status": "REVIEWED",
  "reviewerComment": "同意改进措施，建议下轮持续跟踪。"
}
```

数据库支撑：
- 数据库已支持。
- 当前缺实体字段、VO 返回字段和审核接口。
