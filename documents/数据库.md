# 工程教育专业认证智能服务系统 - 数据库设计说明书

本说明书根据初始化数据库脚本（`init.sql`）整理，旨在详细阐述“工程教育专业认证智能服务系统”（OBE系统）的底层数据模型、表物理结构、表间关系以及核心业务计算逻辑，供系统开发、数据库维护和系统集成时参考。

---

## 1. 概述与设计规范

### 1.1 系统背景
本系统基于成果导向教育（Outcome-Based Education, OBE）理念设计，旨在实现专业培养方案管理、毕业要求分解、课程大纲设计、成绩分维度录入、达成度定量/定性计算、AI持续改进报告生成及佐证材料归档的一体化闭环管理。

### 1.2 数据库环境与参数
*   **适用数据库服务器**：MySQL 8.0 及以上版本
*   **默认字符集**：`utf8mb4`
*   **默认校对规则**：`utf8mb4_unicode_ci`
*   **存储引擎**：`InnoDB` (全面支持事务及外键约束)

---

## 2. 关系实体与逻辑板块划分

本数据库共包含 **16** 张数据表，按照系统业务板块划分为以下 9 个子模块：

1.  **组织与权限模块**：包含组织架构表（`sys_organization`）、系统角色表（`sys_role`）、用户基础信息表（`sys_user`）和用户角色关联表（`sys_user_role`）。
2.  **班级与学生模块**：包含行政班级信息表（`class_info`）和学生拓展信息表（`student_info`）。
3.  **培养方案与指标点**：包含人才培养方案表（`program_scheme`）、毕业要求大项表（`grad_requirement`）和毕业要求二级指标点表（`grad_indicator_point`）。
4.  **课程大纲与OBE矩阵**：包含课程信息表（`course`）、课程目标表（`course_objective`）和课程目标与指标点支撑矩阵（`course_obj_indicator_map`）。
5.  **教学运行与选课**：包含教学班级运行信息表（`teaching_class`）和教学班选课学生关联表（`teaching_class_student`）。
6.  **考核标准与得分录入**：包含课程考核评价方式表（`assessment_method`）、课程考核细小项表（`assessment_item`） and 学生得分明细表（`student_score`）。
7.  **材料归档与直接评价**：包含佐证材料归档表（`evidence_material`）、课程目标直接达成度计算结果表（`course_attainment`）和持续改进分析报告表（`continuous_improvement`）。
8.  **问卷调查与间接评价**：包含调查问卷表（`survey_questionnaire`）和问卷答案表（`survey_answer`）。
9.  **AI基础设施配置**：包含Spring AI智能代理提示词配置表（`ai_prompt_template`）。

## 3. 表物理结构详细设计

### 3.1 组织架构表 (`sys_organization`)
记录学院、专业、行政班级树状组织架构。
*   **主键**：`id` (自增)

| 字段名 | 数据类型 | 约束 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `bigint` | PK, AUTO_INCREMENT | | 主键ID |
| `name` | `varchar(100)` | NOT NULL | | 机构/部门名称 |
| `parent_id` | `bigint` | | NULL | 父级组织ID |
| `type` | `varchar(20)` | NOT NULL | | 级别: `COLLEGE`-学院, `MAJOR`-专业, `CLASS`-行政班 |
| `created_at` | `timestamp` | NOT NULL | `CURRENT_TIMESTAMP` | 创建时间 |

---

## 3.2 系统角色表 (`sys_role`)
维护系统内角色（支持 5 大核心角色代码）。
*   **主键**：`id` (自增)
*   **唯一索引**：`uk_role_code` (`role_code`)

| 字段名 | 数据类型 | 约束 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `bigint` | PK, AUTO_INCREMENT | | 角色ID |
| `role_name` | `varchar(50)` | NOT NULL | | 角色名称 (如：系统管理员) |
| `role_code` | `varchar(50)` | NOT NULL, UNIQUE | | 角色代码: `ADMIN`, `DIRECTOR`, `COORDINATOR`, `INSTRUCTOR`, `STUDENT` |
| `description` | `varchar(200)` | | NULL | 角色职责描述说明 |

---

## 3.3 用户基础信息表 (`sys_user`)
用户核心信息主表，存储登录工号、加密密码及所属组织。
*   **主键**：`id` (自增)
*   **唯一索引**：`uk_username` (`username`)
*   **外键**：`fk_user_org` 指向 `sys_organization(id)`，物理删除组织时用户 org_id 置为 NULL。

| 字段名 | 数据类型 | 约束 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `bigint` | PK, AUTO_INCREMENT | | 用户ID |
| `username` | `varchar(50)` | NOT NULL, UNIQUE | | 登录账号(工号或学号) |
| `password` | `varchar(100)` | NOT NULL | | BCrypt加密密码 |
| `real_name` | `varchar(50)` | NOT NULL | | 真实姓名 |
| `email` | `varchar(100)` | | NULL | 电子邮箱 |
| `phone` | `varchar(20)` | | NULL | 手机号码 |
| `status` | `tinyint` | NOT NULL | `1` | 账号状态: `0`-禁用, `1`-启用 |
| `org_id` | `bigint` | FK | NULL | 所属组织ID |
| `created_at` | `timestamp` | NOT NULL | `CURRENT_TIMESTAMP` | 创建时间 |

---

## 3.4 用户角色关联表 (`sys_user_role`)
维护用户与角色的多对多映射，支持教师身兼多职（如既是授课教师，又是专业负责人）。
*   **联合主键**：`(`user_id`, `role_id`)`
*   **外键**：关联用户表与角色表，级联删除。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `user_id` | `bigint` | PK, FK | 关联用户ID |
| `role_id` | `bigint` | PK, FK | 关联角色ID |

---

*   **外键**：`fk_class_major` 指向 `sys_organization(id)` (级联删除)

| `major_id` | `bigint` | FK | 所属专业ID |
| `grade` | `int` | NOT NULL | 招收年级 (如：`2024`) |

| :--- | :--- | :--- | :--- |

| `student_no` | `varchar(30)` | NOT NULL, UNIQUE | 唯一学号 |
| `class_id` | `bigint` | FK | 关联的行政班级ID |

## 3.8 毕业要求大项表 (`grad_requirement`)

*   **主键**：`id` (自增)
*   **外键**：`fk_gr_scheme` 关联 `program_scheme(id)`。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |

| `content` | `text` | NOT NULL | 指标点具体的达成标准描述 |

| `id` | `bigint` | PK, AUTO_INCREMENT | 课程主键ID |
| `course_code` | `varchar(50)` | NOT NULL, UNIQUE | 课程编码 (如: `SE-302`) |

---

*   **主键**：`id` (自增)
*   **外键**：`fk_co_course` 关联 `course(id)`。

| 字段名 | 数据类型 | 约束 | 描述 |

---

## 3.12 课程目标与指标点支撑矩阵 (`course_obj_indicator_map`)
配置各课程目标支撑二级指标点的支撑度权重，系统计算最终达成度时的核心系数基础。
*   **主键**：`id` (自增)
*   **唯一索引**：`uk_co_indicator` (`course_objective_id`, `indicator_point_id`)，防止同一关联配置重复录入。
*   **外键**：关联 `course_objective` 与 `grad_indicator_point` (级联删除)。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `bigint` | PK, AUTO_INCREMENT | 映射关联ID |
| `course_objective_id` | `bigint` | FK | 课程目标ID |
| `indicator_point_id` | `bigint` | FK | 二级指标点ID |
| `weight` | `decimal(3,2)` | NOT NULL | 支撑权重系数 (限制在 `0.01 - 1.00`) |

---

*   **主键**：`id` (自增)
*   **外键**：`fk_am_course` 关联 `course(id)`。

| 字段名 | 数据类型 | 约束 | 描述 |

| `name` | `varchar(50)` | NOT NULL | 考核版块名称 (如: 期末考试, 实验项目) |

*   **主键**：`id` (自增)
*   **外键**：关联 `assessment_method` 和 `course_objective`。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `bigint` | PK, AUTO_INCREMENT | 考核小项主键ID |
| `method_id` | `bigint` | FK | 关联的父考核构成ID |
| `name` | `varchar(100)` | NOT NULL | 细项名称 (如：试卷第1题(软件过程题)) |
| `max_score` | `decimal(5,2)` | NOT NULL | 该小项满分数值 (如：`20.00`) |
| `course_objective_id` | `bigint` | FK | 强绑定的唯一课程目标ID (OBE核心关系) |

## 3.17 学生得分明细表 (`student_score`)

*   **主键**：`id` (自增)
*   **唯一索引**：`uk_student_item_score` (`student_id`, `assessment_item_id`)，同一个学生的一道考核题只能有一个得分记录。
*   **外键**：关联 `sys_user` 与 `assessment_item`。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `bigint` | PK, AUTO_INCREMENT | 得分记录主键ID |
| `student_id` | `bigint` | FK | 学生用户ID |
| `assessment_item_id` | `bigint` | FK | 对应考核项的ID |
| `actual_score` | `decimal(5,2)` | NOT NULL | 该学生该考核题的实测得分值 |

## 4. OBE 核心达成度计算逻辑说明

数据库设计通过 `assessment_method` -> `assessment_item` -> `student_score` 的层级结构，实现了直接评价法下课程目标达成度的精确计算。其数学公式和逻辑如下：

### 4.1 课程目标达成度直接评价公式
对于某一课程目标 $CO_k$，其达成度 $A(CO_k)$ 计算方式为：

$$A(CO_k) = \frac{\sum (考核项分权重 \times 班级该项绑定 CO_k 题目的平均分)}{\sum (考核项分权重 \times 该项绑定 CO_k 题目的总满分)}$$

在数据库中的具体组装查询逻辑：
1.  **确定权重关系**：
    查询 `assessment_method.weight`（设为 $W_m$）。
2.  **筛选考核项与实际分**：
    定位至 `assessment_item`（设其 $max\_score$ 为 $S_{max}$），并匹配到对应 $CO_k$。
3.  **计算平均得分**：
    对选修了该 `teaching_class` 的所有学生，计算其在 `student_score.actual_score` 上的平均分（设为 $S_{avg}$）。
4.  **汇总计算**：
    针对指定课程目标，遍历所有关联小项进行公式计算，将计算结果保存于 `course_attainment` 固化表中，当低于设定的警戒线（一般为 `0.680`）时，在前端触发持续改进模块（`continuous_improvement`）的编写提醒。
